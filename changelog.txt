v0.3.5 (2026-06-05)
--------------------------
Bug Fixes:
  - Fixed: Kijkwijzer scraper blocked by Akamai bot detection on hosts with OpenSSL 3.0 SECLEVEL=2; the restricted 17-cipher TLS handshake triggered a JS challenge. Added a custom HTTPAdapter that sets SECLEVEL=1 to restore the full ~60-cipher set.
  - Fixed: BBFC and Kijkwijzer canaries matched the wrong BBFC entry for Interstellar (rated 15 instead of 12) because no year filter was applied. Added year=2014 to both canary definitions.
  - Fixed: Canary failures only logged warnings but did not disable the broken scraper for the scan cycle. A mismatch, no-result, or exception now removes that scraper from the enabled set for the current run.
  - Fixed: Manual scan (RunScript) skipped canary checks entirely, running scrapers unchecked. Both entry points now run canaries before scanning.

Improvements:
  - All four scrapers (kijkwijzer, bbfc, fsk, medieraadet) use a shared create_scraper_session() factory with browser-like TLS settings, with graceful fallback on unsupported platforms.

Internal:
  - 19 new tests covering canary pass/fail/error/skip/passthrough scenarios, TLS adapter behavior, and session factory wiring across all providers.

v0.3.4 (2026-06-05)
--------------------------
Improvements:
  - Scraper title matching now uses accent normalization as fallback (Pokémon matches Pokemon)
  - Separator-aware prefix matching: titles with colon, dash, or space suffixes now match (e.g. "Pokémon: XY" matches search for "Pokémon")
  - Shared title_matches() utility used by all four scrapers for consistent behavior
  - BBFC now supports prefix matching (previously exact-only)
  - FSK year filter skipped for TV shows; FSK classifies seasons years after air date, causing false negatives for older series

v0.3.3 (2026-06-05)
--------------------------
Improvements:
  - Every per-item log line now includes title= for greppable debug logs
  - Standardized provider outcome messages: "Match found", "No title match", "No results" across all scrapers
  - All scraper match and no-match log lines now include result_count=
  - Kijkwijzer warning logs now include title=
  - Inference chain debug logs now include title=
  - "Item pending" log now includes media_type=

v0.3.2 (2026-06-05)
--------------------------
Improvements:
  - Native scraper (FSK, BBFC, Medieraadet, Kijkwijzer) now runs before TMDB inference for your own country
  - Inference chain walks countries by cultural relevance, checking TMDB then the scraper for each country before moving on
  - Scrapers fill gaps where TMDB is missing certifications for culturally close countries

Fixes:
  - "Replace incorrect certifications" now enforces the configured prefix: items with the wrong prefix, a missing prefix, or an unwanted prefix are flagged and re-rated
  - Blank prefix override correctly replaces prefixed values like "NL:12" with bare "12"
  - "Rated NR" is now replaced when it does not match the expected prefix format

v0.3.1 (2026-06-04)
--------------------------
Improvements:
  - Certification prefix now defaults to the country preset value (e.g. NL: for Netherlands, Rated for US) instead of being empty
  - Added "Override certification prefix" toggle (advanced setting) to use a custom prefix
  - The prefix text field only appears when the override toggle is enabled

v0.3.0 (2026-06-04)
