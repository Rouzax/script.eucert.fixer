v0.3.3 (2026-06-05)
--------------------------
Improvements:
  - Every per-item log line now includes title= for greppable debug logs
  - Standardized provider outcome messages: "Match found", "No title match", "No results" across all scrapers
  - All scraper match and no-match log lines now include result_count=
  - Kijkwijzer warning logs now include title=
  - Inference chain debug logs now include title=
  - "Item pending" log now includes media_type=

v0.3.2 (2026-06-05)
--------------------------
Improvements:
  - Native scraper (FSK, BBFC, Medieraadet, Kijkwijzer) now runs before TMDB inference for your own country
  - Inference chain walks countries by cultural relevance, checking TMDB then the scraper for each country before moving on
  - Scrapers fill gaps where TMDB is missing certifications for culturally close countries

Fixes:
  - "Replace incorrect certifications" now enforces the configured prefix: items with the wrong prefix, a missing prefix, or an unwanted prefix are flagged and re-rated
  - Blank prefix override correctly replaces prefixed values like "NL:12" with bare "12"
  - "Rated NR" is now replaced when it does not match the expected prefix format

v0.3.1 (2026-06-04)
--------------------------
Improvements:
  - Certification prefix now defaults to the country preset value (e.g. NL: for Netherlands, Rated for US) instead of being empty
  - Added "Override certification prefix" toggle (advanced setting) to use a custom prefix
  - The prefix text field only appears when the override toggle is enabled

v0.3.0 (2026-06-04)
--------------------------
  - Initial release
  - Automatically fills missing age certifications for movies and TV shows
  - 9 country presets: NL, BE, DE, AT, GB, US, FR, DK, SE
  - Searches TMDB, OMDB, and national rating authority websites
  - FSK (Germany), BBFC (UK), Medieraadet (Denmark), Kijkwijzer (Netherlands)
  - Converts between rating systems using conservative rounding
  - Per-scraper enable/disable toggles
  - Runs as a background service with configurable scan intervals
  - Manual scan trigger from Programs menu
  - Retry tracking for items not yet in any database
  - Structured debug logging to separate file
